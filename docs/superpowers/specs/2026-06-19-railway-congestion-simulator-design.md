# 철도역사 혼잡도 합성데이터 생성 시뮬레이터 — 설계 문서

- 작성일: 2026-06-19
- 목적: 철도역사 내부 주요 지점의 혼잡도(공간 내 인원수) 시계열을 현실적으로 생성하여, GNN(STGCN 등) 기반 혼잡도 예측 모델의 학습/평가/테스트 데이터셋을 만드는 시뮬레이터를 개발한다.

## 0. 확정된 핵심 결정 (브레인스토밍 결과)

| # | 결정사항 | 선택 |
|---|---|---|
| 1 | 배포·실행 아키텍처 | **웹 + Pyodide** (GitHub Pages 정적 배포, Python 코어를 브라우저 WASM에서 실행) |
| 2 | 시스템 이탈(sink) 모델 | **출입구 = 이탈(exit), 승강장 = 탑승(boarding)** — source이면서 동시에 sink |
| 3 | 데이터 확률성 | **연속(실수) 결정론 계산 + 선택적 노이즈** (발생은 확률분포, 관측노이즈·결측 옵션) |
| 4 | 실행 방식 | **웹 인터랙티브 중심** + 브라우저 내 N회 일괄 실행(배치) → ZIP 다운로드 |
| 5 | 승강장 열차 스케줄 | **주기적 배차 + 지터** (첫 도착·headway·정규 지터·하차인원 분포) |
| 6 | 탑승 규칙 | **열차 정원 기반 부분 탑승** (정원 초과분은 다음 열차 대기 → 혼잡 누적/해소) |
| 7 | 혼잡→체류확률 모델 | **보행자 기본도(Weidmann/Kladek) 기반** 속도-밀도 관계 |

### 원문 요구사항의 공식 오류 정정 (중요)
- 원문 .txt: `N(t+1) = N(t)·체류확률 + 유입 − N(t)·이동확률`
- 본 설계(정정): `N(t+1) = N(t)·체류확률 + 유입 + 자체발생`
- 이유: `체류확률 = 1 − 이동확률`이므로 `N(t)·체류확률`에는 이미 유출분이 제외되어 있다. 원문은 유출을 한 번 더 빼는 **이중 차감 오류**가 있다. 링크 소요시간(travel time)을 반영한 도착(유입)을 **지연 버퍼**로 처리하면 시스템 전체(노드 + 링크 이동중) 인원이 보존된다.

## 1. 전체 아키텍처

GitHub Pages(정적 호스팅, 서버 불가) + 핵심 알고리즘 Python이라는 두 제약을 **Pyodide**(CPython+numpy를 WebAssembly로 브라우저에서 실행)로 동시 충족한다.

```
[브라우저 / GitHub Pages 정적 사이트]
 ├─ React UI (메인 스레드)
 │   ├─ GraphEditor   : 노드-링크 GUI 입력 (React Flow)
 │   ├─ Inspector     : 노드/링크 속성 편집 + 실시간 검증
 │   ├─ SimControls   : 총 시간 / Δt / 배속 / 시드 / 시작·정지·스텝·리셋
 │   ├─ Dashboard     : 실시간 혼잡도 라인차트 + 그래프 히트맵 (Plotly)
 │   ├─ BatchPanel    : N회 일괄 실행 + 파라미터 변주 → ZIP 다운로드
 │   └─ ExportPanel   : CSV / 설정 JSON / GNN(STGCN)용 행렬 번들
 │
 └─ Web Worker (UI 블로킹 방지)
     └─ Pyodide → Python 시뮬레이션 코어 (numpy)
         model.py / generation.py / pedestrian.py / engine.py / io.py
```

- **Python 코어**는 순수 Python+numpy로 작성 → Pyodide에서 그대로 실행되며, **동일 코드를 로컬에서 pytest로 단위테스트**할 수 있다(개발/검증용, 별도 배포물 아님).
- 시뮬레이션은 **Web Worker**에서 실행하여 메인 스레드(UI)를 막지 않는다. 제어 메시지(시작/일시정지/배속/스텝/리셋)와 결과 메시지(주기적 혼잡도 스냅샷, 진행률)를 주고받는다.
- Pyodide 자산은 CDN(jsdelivr)에서 로드(첫 로딩 시 수 MB 다운로드, 이후 캐시). 배포는 Vite 빌드 → GitHub Pages(GitHub Actions).

### Python 코어 모듈 구성
| 모듈 | 책임 | 주요 인터페이스 |
|---|---|---|
| `model.py` | 데이터 구조(Node/Link/StationGraph/SimConfig), 검증 규칙 | `StationGraph.from_json()`, `.validate()` |
| `generation.py` | 발생 프로세스(Poisson/정규펄스/상수/열차버스트) | `Generator.sample(t, rng)` |
| `pedestrian.py` | 보행자 기본도(Weidmann) 속도-밀도, 밀도→이동확률 | `speed_ratio(density, params)` |

| `initial_population` | float | 초기 인원 N(0), 기본 0 |

| `exit_weight` | float[0,1] | (출입구 등) 시스템 이탈 sink 가중치 **[추가]**. 실제 출력링크 가중치 합 + exit_weight = 1 |
| `platform` | obj? | (승강장만) 열차 스케줄/정원/하차 분포 |

## 3. 시뮬레이션 알고리즘 (핵심)

상태:
- 각 노드 i의 인원 `Nᵢ(t)` (연속 실수 ≥ 0)
- 각 링크 (i→j, 소요시간 τ)의 **지연 버퍼** `B_{ij}` (길이 τ): t에 진입한 인원이 t+τ에 도착

이산 시간 `t → t+1` 매 스텝 절차:

**① 혼잡도 기반 동적 체류/이동 확률**
- 밀도 `ρᵢ = Nᵢ(t) / areaᵢ`
- 속도비 `fᵢ = v(ρᵢ) / v_free` (Weidmann, 0~1; `congestion_enabled=false`면 fᵢ=1)
- `이동확률ᵢ = (1 − base_stayᵢ) · fᵢ`
- `체류확률ᵢ = 1 − 이동확률ᵢ` → 혼잡할수록 이동확률↓, 체류↑

**② 유출 계산**
- 이동인원 `moversᵢ = Nᵢ(t) · 이동확률ᵢ`
- 링크별 유출 `flow(i→j) = moversᵢ · w(i→j)` ← 요구사항 그대로
- 이탈(sink) `exitᵢ = moversᵢ · exit_weightᵢ` (시스템에서 영구 제거, 누적이탈로 집계)

**③ 인원 갱신 (정정된 공식)**
```
Nᵢ(t+1) = Nᵢ(t)·체류확률ᵢ  +  Σ_{k→i} arrive_{ki}(t)  +  발생ᵢ(t)
            └ 잔류(stayers)      └ 링크 만기 도착(유입)      └ 자체발생
```
- `Nᵢ(t)·체류확률ᵢ = Nᵢ(t) − moversᵢ` (잔류 = 전체 − 이동)
- `arrive_{ki}(t) = B_{ki}[0]` (이번 스텝에 만기 도착하는 코호트)

**④ 링크 지연 버퍼 진행**
- 만기분 `B_{ij}[0]`을 도착 유입으로 소비
- 시프트: `B_{ij}[k] ← B_{ij}[k+1]` (k < τ−1), `B_{ij}[τ−1] ← flow(i→j)`
- ⇒ t에 진입 → t+τ 도착 (예: τ=3이면 t+3 도착, 요구사항 예시 일치)

**⑤ 승강장 열차 이벤트** (해당 스텝이 열차 도착 시각이면)
- **탑승 먼저**: `boarded = min(열차정원, Nₚ(t))` 만큼 시스템 이탈(sink). 정원 초과분은 그대로 남아 다음 열차까지 누적
- **하차 나중**: 하차인원 버스트를 `발생ₚ(t)`로 추가
- "탑승 → 하차" 순서로, 방금 하차한 사람이 같은 열차에 즉시 재탑승되는 오류 방지
- 승강장도 일반 노드처럼 출력링크(하차객이 출구 방향으로 이동)와 체류확률을 가진다

**⑥ 기록**: 모든 노드의 `Nᵢ(t+1)`을 혼잡도 시계열로 기록. (선택) 관측노이즈 적용본도 별도 기록 가능.

### 질량 보존
시스템 총량 = `Σ Nᵢ + Σ 버퍼 내 이동중 인원`. 매 스텝 `+ 발생(source) − 이탈(출입구 exit + 승강장 탑승)` 외에는 보존된다. → 단위테스트로 검증.

### 확률성 처리
- 기본은 연속 실수 결정론 계산(매끈, 재현 용이).
- `stochastic=true`면 **발생**을 분포에서 표본추출(예: Poisson(λ·Δt)). 동일 `seed` → 동일 결과.
- (선택) 관측노이즈: 기록 시 가우시안/포아송 노이즈, 무작위 결측(NaN) 삽입 — AI 강건성 학습용.

## 4. 현실성 모델

### 4.1 보행자 기본도 (Weidmann/Kladek)
```
v(ρ) = v_free · [ 1 − exp( −γ · (1/ρ − 1/ρ_max) ) ]   (ρ < ρ_max),   ρ ≥ ρ_max 면 v=0
f(ρ) = v(ρ) / v_free  ∈ [0,1]
```
- 기본값: `v_free = 1.34 m/s`, `ρ_max = 5.4 인/m²`, `γ = 1.913`. 노드별 조정 가능.
- 의미: 임계밀도에서 유량(`q = ρ·v`)이 최대→감소하므로 **병목 큐가 자연 발생**(별도 용량 파라미터 없이도 현실적 정체 표현).

### 4.2 발생 분포
- **출입구**: Poisson(시간가변 λ(t) — 출근/퇴근 피크 프로파일 프리셋 제공) / 정규 펄스(이벤트성 급증) / 상수 중 선택.
- **승강장**: 주기적 배차(첫 도착시각, headway, 정규 지터 σ) + 하차인원(상수 / Poisson(평균) / 정규(평균,표준편차)).

### 4.3 노드 종류별 기본 프로필
| 종류 | v_free(m/s) | 혼잡모델 | 비고 |
|---|---|---|---|
| 통로 | 1.34 | on | 표준 보행 |
| 계단 | 0.6~0.7 | on | 상/하행 느림 |
| 에스컬레이터 | 고정 | off | 고정 속도 운송, 소요시간은 링크로 |
| 엘리베이터 | 고정 | off | 배치성 운송(고급: 용량 옵션) |
| 게이트 | 1.0 | on | 통과 지점, 병목 발생 쉬움 |
| 출입구 | 1.34 | on | source + exit sink |
| 승강장 | 1.20 | on | source(하차) + boarding sink |

### 4.4 (선택) 고급 현실성 옵션 (기본 off)
- 노드/링크 최대 유량(용량) 제한 → 명시적 큐잉.
- 측정 노이즈/결측 → 센서 데이터 모사.

### 4.5 모델링 가이드 (거시적 유량 모델의 한계 보완)
- 본 모델은 요구사항에 따른 **거시적 유량(population-flow) 모델**이다(개별 OD 경로 추적 없음).
- 단일 인원 풀이므로 "들어오는 사람"과 "나가는 사람"의 의도가 한 노드에서 섞일 수 있다. 깨끗한 분리가 필요하면 **출입구를 입구용/출구용 노드로 분리**하여 모델링하도록 가이드 제공(입구: 발생+내부 링크 / 출구: exit_weight=1).

## 5. GUI / 대시보드 / 제어

### 5.1 GraphEditor (React Flow)
- 종류별 노드 팔레트에서 드래그하여 노드 생성, 노드 간 링크 드로잉.
- 노드/링크 클릭 시 Inspector 패널에서 속성 편집.
- **실시간 검증**: 체류+이동=1, 출력링크 가중치 합=1(정규화 버튼 제공), 발생노드 규칙(출입구·승강장만 발생), 면적>0, 거리>0, 연결성/도달성 경고(이탈 경로 없는 source는 인원 누적 경고).

### 5.2 설정 저장/불러오기
- 역 구성 + 파라미터를 JSON으로 export/import.
- localStorage 자동저장(작업 중 보존).
- **예제 템플릿** 제공: 소형 역 샘플(빠른 시작용) **[추가]**.

### 5.3 SimControls
- 총 시뮬레이션 시간, Δt, **배속(X배속)**, 시작/일시정지/스텝/리셋, 랜덤 시드, 확률모드 on/off, (선택) 노이즈 설정.
- 배속: Worker가 목표 steps/sec로 throttle 실행하며 주기적 스냅샷 전송. "즉시 실행(애니메이션 생략)" 모드 별도 제공.

### 5.4 Dashboard (실시간)
- 노드별 혼잡도 시계열 라인차트(노드 선택).
- 그래프 위 노드 색 = 현재 혼잡도 히트맵(실시간 갱신).
- 집계 지표: 총 재실 인원, 누적 발생, 누적 이탈, 종류별 합계.
- 진행률 바, 현재 시뮬레이션 시각.

## 6. 출력 (Export)

### 6.1 혼잡도 시계열 CSV

## 8. 테스트 전략 (TDD)

순수 Python 코어를 pytest로 우선 검증(테스트 우선):
- 질량 보존(발생−이탈 외 보존).
- 결정론적 유량 정확성(`flow = N·이동확률·가중치`).
- 소요시간 지연 정확성(τ 스텝 후 도착).
- 발생 분포 통계(대수의 법칙 범위).
- Weidmann 단조성(밀도↑ → 속도↓ → 체류↑).
- sink 동작(출입구 이탈, 승강장 정원 기반 부분탑승, 탑승→하차 순서).
- 검증 규칙(가중치 합, 체류+이동=1).
- 손계산 골든 시나리오(소형 그래프 → 기대 시계열 일치).

프런트엔드: 검증 로직/배치 매니페스트 등 핵심 단위테스트 + 스모크 테스트(앱 로드, 샘플 실행).
