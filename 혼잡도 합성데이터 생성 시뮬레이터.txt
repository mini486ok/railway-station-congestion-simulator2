철도역사 내부 주요 지점의 혼잡도(해당 공간에 존재하는 인원수)에 대한 시계열 데이터가 있을때, 이를 기반으로 향후 혼잡도를 예측하는 GNN(STGCN 등) 기반 혼잡도 예측 AI 모델을 개발하려고 해. 해당 AI 모델을 학습, 평가, 테스트 할 수 있는 합성데이터를 생성하는 시뮬레이터를 하기 요구사항을 반영해서 만들 수 있는 계획을 수립해줘. (코딩은 아직 시작하지말고, 계획에 대해서 내가 최종 컨펌하면 그때 개발을 시작해줘.)
- 전체 시스템은 이산 시간으로 돌아간다고 가정 
- 철도역사 내부 주요지점을 모사하기 위해, 철도역사 내부의 주요지점, 연결성 등을 노드-링크로 직접 입력하여 대상을 구축할 수 있도록 하는 GUI 기반 입력 기능 필요
- 노드는 노드명, 종류, 체류 확률, 이동 확률 등 속성 필요(혹시 더 필요하면 추가해도 됨)
- 노드 종류는 출입구, 통로, 계단, 에스컬레이터, 엘리베이터, 게이트, 승강장 등이 될 수 있으며, 이 중에 선택할 수 있어야 함
- 특정 노드에 대해 다음 시점에도 해당 노드에 있을 확률을 의미하는 "체류 확률"과 다른 노드로 이동할 확률을 의미하는 "이동 확률"의 합은 1이어야 함
- 링크는 입력노드, 출력노드, 노드 간 거리, 가중치, 소요시간(이산 시간 기준 단위로 환산하여 반영, 예를 들어 소요시간이 3이면 입력노드에서 t시점에 있던 이용자가 출력노드에는 t+3 시점에 도착, 노드 간 거리와 평균 이용자 보행속력 및 이산 시간 기준 단위를 알면 자동으로 계산 가능) 등 속성 필요(혹시 더 필요하면 추가해도 됨)
- 특정 노드에 (t+1) 시점에 존재하는 인원 수는 "t시점에 존재했던 인원 수*체류 확률 + 새롭게 입력(유입)되는 이용자 수 - t시점에 존재했던 인원 수*이동 확률"이어야 함
- 특정 노드에 대해 출력 링크가 여러 개인 경우, 해당 링크들의 가중치 합은 1이어야 함
- 노드 A에서 노드 B로 출력(유출)되는 이용자 수는 "노드 A에 존재하는 인원 수*이동 확률*노드 A에서 노드 B로 이동하는 링크의 가중치 값"임
- 다른 노드에서 이동해오는 이용자가 아닌, 해당 노드에서 이용자가 자체 발생하는 노드의 종류는 출입구, 승강장만 존재하며, 노드 종류가 출입구, 승강장인 경우에는 해당 노드에서 새롭게 생성되는 이용자 발생 패턴을 설정할 수 있는 기능 필요(프아송 분포, 정규 분포 등 합리적인 분포를 제시하여 선택할 수 있는 기능이 필요하며, 승강장의 경우에는 열차가 도착하는 시점에만 순간적으로 이용자들이 한 번에 발생하는 패턴으로 설정할 수 있는 기능 필요)
- 각각의 노드에 대해서 면적 대비 존재하는 인원 수(혼잡도)에 따라 체류 확률이 동적으로 변경될 수 있도록 설정하는 기능 필요(혼잡도가 높으면 보행 속력이 느려져 해당 노드에 체류하는 시간이 길어질 것이므로)
- 시뮬레이션 시간을 설정할 수 있는 기능, 시뮬레이션 속도를 설정할 수 있는 기능(X배속 등), 시뮬레이션 중 각 노드의 혼잡도 그래프 등 실시간 정보를 조회할 수 있는 대시보드 기능, 시뮬레이션 종료 후 시뮬레이션 설정 파라미터 및 각 노드에 대한 혼잡도 시계열 데이터를 csv 파일 형태로 출력해주는 기능 등 필요
- 만약 가능하다면, 누구나 어디서든지 활용할 수 있도록 웹 기반 시뮬레이터가 될 수 있도록 해줘.
- 웹 기반 시뮬레이터의 경우, github pages를 통해 배포할 예정이니, github pages를 통해 배포가 가능하도록 개발해줘.
- 웹 기반 시뮬레이터가 어렵다면, 로컬 PC에서 exe 파일로 실행할 수 있도록 해줘.
- 전반적으로 핵심 시뮬레이션 알고리즘은 파이썬 언어를 주로 사용하도록 해줘. 
- 위에서 정의한 요구사항 외에도, 철도역사 주요 지점의 혼잡도 시계열 데이터를 이용자의 보행 특성/모델, 물리적 사실 등을 반영하여 현실과 최대한 유사하게 합리적으로 생성할 수 있는 시뮬레이터가 되도록 여러가지를 고려해서 기능을 개발해줘.
